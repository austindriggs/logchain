## INTRODUCTION

We settled down onto a topic and really defined the scope better:
- Our app, LogChain, continuously monitors Docker container logs, cryptographically chains each log entry to preserve an immutable record, and alerts administrators when suspicious activity or log tampering is detected.

A "historian" is a broad word with many different off the shelf solutions and meanings. Our application will focus on:
- Lightweight for small servers and homelabs: Will be free and open sourced for anyone to use.
- Flexible because its written in python and runs in docker, allowing for the small servers to customize it through programming themselves or a config file.
- It will have a notification service to not only securly maintain the logs, but alert admins if it detects tampering.


## R&D

While doing research, we found no other solutions that were: for small servers, written in python and ran using docker, AND where the documentation was written in english. That's what makes our solution unique.

Some of the research that we did includes:
- Logs: How docker stores logs and how to read them for each container.
- Chain: Researching different algorithms and learning how feasible this is to do in real time.
- App: Flask is simple enough to use (minor learning curve) and is written in python.
- Alerts: ntfy is a popular choice for small, self-hosted servers.

Some of the development that we did includes:
- Setup a github repo to host our code and collaborate.
- Learned how to setup a `Dockerfile`, `docker-compose.yml` (and commands), and python dependancies in WSL/Ubuntu.
- Learned how to access the docker log socket in `/var/run/docker.sock` and serve them on Flask for all containers.


## CONCLUSION

Some of our next steps include:

- Actually hashing the logs and figuring out the best algorithm and timing for it.
